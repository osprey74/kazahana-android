# Phase 4 テスト項目

## 1. テーマ切り替え (ライト / ダーク / システム連動)

### 1-1. テーマ選択
- [ ] 設定画面で「システム」チップをタップ → 端末のダーク/ライト設定に追従すること
- [ ] 設定画面で「ライト」チップをタップ → アプリ全体がライトテーマに切り替わること
- [ ] 設定画面で「ダーク」チップをタップ → アプリ全体がダークテーマに切り替わること
- [ ] ステータスバーのアイコン色がテーマに応じて切り替わること (ライト時: 暗色アイコン、ダーク時: 明色アイコン)

### 1-2. 永続化
- [ ] テーマを「ダーク」に変更してアプリを再起動 → ダークテーマで起動すること
- [ ] テーマを「ライト」に変更してアプリを再起動 → ライトテーマで起動すること
- [ ] テーマを「システム」に戻してアプリを再起動 → 端末設定に追従すること

### 1-3. 画面遷移
- [ ] テーマ切替後、各画面 (ホーム/検索/通知/メッセージ/プロフィール) に遷移して表示崩れがないこと
- [ ] テーマ切替後、投稿作成画面・スレッド画面で配色が正しいこと

## 2. 多言語対応 (11言語)

### 2-1. アプリ内言語切替 (設定画面ドロップダウン)
- [ ] 設定画面の「言語」セクションにドロップダウンが表示されること
- [ ] ドロップダウンに12項目 (システム + 11言語) が表示されること
- [ ] 各言語がネイティブ名で表示されること (例: 日本語、Deutsch、한국어)
- [ ] 「システム」を選択 → 端末の言語設定に追従すること
- [ ] 「日本語」を選択 → 端末言語に関係なくアプリUIが即座に日本語に切り替わること
- [ ] 「English」を選択 → 端末言語に関係なくアプリUIが即座に英語に切り替わること
- [ ] 「Deutsch」を選択 → アプリUIがドイツ語に切り替わること
- [ ] 「Español」を選択 → アプリUIがスペイン語に切り替わること
- [ ] 「Français」を選択 → アプリUIがフランス語に切り替わること
- [ ] 「Bahasa Indonesia」を選択 → アプリUIがインドネシア語に切り替わること
- [ ] 「한국어」を選択 → アプリUIが韓国語に切り替わること
- [ ] 「Português」を選択 → アプリUIがポルトガル語に切り替わること
- [ ] 「Русский」を選択 → アプリUIがロシア語に切り替わること
- [ ] 「简体中文」を選択 → アプリUIが簡体中国語に切り替わること
- [ ] 「繁體中文」を選択 → アプリUIが繁体中国語に切り替わること

### 2-2. アプリ内言語設定の優先度
- [ ] アプリ内で言語を設定 → 端末の言語設定を変更してもアプリ内設定が優先されること
- [ ] アプリ内で「システム」に戻す → 端末の言語設定に追従すること

### 2-3. 永続化
- [ ] 言語を変更してアプリを再起動 → 選択した言語が保持されていること
- [ ] 言語を変更してアプリをキルして再起動 → 選択した言語が保持されていること

### 2-4. フォールバック
- [ ] 端末を未対応言語 (例: タイ語) に設定 + アプリ内「システム」 → 英語でフォールバック表示されること

### 2-5. 文字列確認 (各言語で以下を目視チェック)
- [ ] ボトムナビゲーションの5タブ名が翻訳されていること
- [ ] ログイン画面のラベル・ボタンが翻訳されていること
- [ ] 設定画面のセクションタイトル・チップ・ドロップダウンが翻訳されていること
- [ ] 通報ダイアログの理由選択肢が翻訳されていること
- [ ] 文字列が途切れず表示されていること (特にドイツ語・ロシア語の長い単語)

## 3. コンテンツモデレーション (ラベル判定 / フィルタ / ブラー)

### 3-1. 設定画面
- [ ] 「成人向けコンテンツを有効にする」トグルのON/OFFが動作すること
- [ ] トグルOFF時 → ヌード/性的/ポルノの3項目が非表示になること
- [ ] トグルON時 → ヌード/性的/ポルノの3項目が表示されること
- [ ] 「過激なメディア」は成人向けトグルに関係なく常に表示されること
- [ ] 各ラベルについて非表示/警告/表示の3択が切替できること
- [ ] 設定変更がアプリ再起動後も保持されること

### 3-2. ラベル判定ロジック
- [ ] `porn` ラベル付き投稿 + 成人向けOFF → 投稿が非表示 (HIDE) になること
- [ ] `porn` ラベル付き投稿 + 成人向けON + ポルノ=警告 → 警告オーバーレイが表示されること
- [ ] 警告オーバーレイをタップ → 投稿内容が展開表示されること
- [ ] `porn` ラベル付き投稿 + 成人向けON + ポルノ=表示 → 通常通り表示されること
- [ ] `graphic-media` ラベル付き投稿 + 過激メディア=非表示 → 非表示になること
- [ ] `graphic-media` ラベル付き投稿 + 過激メディア=警告 → 警告オーバーレイ表示
- [ ] `neg: true` のラベル → モデレーション判定をスキップすること
- [ ] ラベルなしの通常投稿 → モデレーションの影響を受けないこと

### 3-3. 表示対象
- [ ] ホームタイムラインの投稿にモデレーションが適用されること
- [ ] 検索結果の投稿にモデレーションが適用されること
- [ ] プロフィール画面の投稿にモデレーションが適用されること
- [ ] 通知画面内の引用投稿にモデレーションが適用されること

## 4. 通報機能

### 4-1. 通報ダイアログUI
- [ ] 通報ダイアログに6つの理由 (スパム/規約違反/誤解を招く/性的/嫌がらせ/その他) が表示されること
- [ ] 理由をラジオボタンで1つ選択できること
- [ ] 詳細テキスト入力欄が表示されること
- [ ] 理由未選択時は「通報する」ボタンが無効であること
- [ ] 「キャンセル」ボタンでダイアログが閉じること

### 4-2. 投稿通報
- [ ] 投稿に対する通報 → `com.atproto.moderation.createReport` に `com.atproto.repo.strongRef` が送信されること
- [ ] 通報送信中にローディング表示されること
- [ ] 通報成功 → 成功メッセージが表示されること
- [ ] 通報失敗 → エラーメッセージが表示されること

### 4-3. ユーザー通報
- [ ] ユーザーに対する通報 → `com.atproto.moderation.createReport` に `com.atproto.admin.defs#repoRef` が送信されること
- [ ] 詳細入力なしでも通報可能なこと
- [ ] 詳細入力ありの場合、`reason` フィールドが送信されること

## 5. ダイレクトメッセージ (会話一覧 / 送受信)

### 5-1. 会話一覧
- [ ] メッセージタブをタップ → 会話一覧が表示されること
- [ ] 各会話に相手のアバター・表示名・最新メッセージ・時刻が表示されること
- [ ] 未読件数がバッジで表示されること
- [ ] 会話が存在しない場合 → 「会話はありません」と表示されること
- [ ] Pull-to-Refreshで会話一覧が更新されること
- [ ] 削除済みメッセージが最新の場合 → 「メッセージが削除されました」と表示されること

### 5-2. チャット画面遷移
- [ ] 会話行をタップ → チャット画面に遷移すること
- [ ] チャット画面のヘッダーに相手のユーザー名が表示されること
- [ ] 戻るボタンで会話一覧に戻れること
- [ ] ボトムナビゲーションとFABが非表示になること

### 5-3. メッセージ表示
- [ ] 自分のメッセージ → 右寄せのバブルで表示されること
- [ ] 相手のメッセージ → 左寄せのバブルで表示されること
- [ ] メッセージの送信時刻が表示されること
- [ ] 削除済みメッセージ → 「メッセージが削除されました」テキストで表示されること
- [ ] 最新メッセージが画面下部 (最初に見える位置) にあること

### 5-4. メッセージ送信
- [ ] テキスト入力欄にメッセージを入力できること
- [ ] 空白のみの入力では送信ボタンが無効であること
- [ ] 送信ボタンタップ → メッセージが送信され、入力欄がクリアされること
- [ ] 送信したメッセージが即座にチャット画面に表示されること
- [ ] 送信中はローディング表示されること

### 5-5. ページネーション
- [ ] 古いメッセージへスクロール → 追加メッセージが読み込まれること
- [ ] 会話一覧の下方スクロール → 追加の会話が読み込まれること

### 5-6. エラー処理
- [ ] DM権限のないアプリパスワード使用時 → エラーメッセージが表示されること
- [ ] ネットワークエラー時 → エラーメッセージが表示されること

### 6-1. 画面遷移

- [ ] 他人のプロフィール画面では歯車アイコンが表示されないこと
- [ ] 歯車アイコンタップ → 設定画面に遷移すること
- [ ] 設定画面の戻るボタン → プロフィール画面に戻ること
- [ ] ボトムナビゲーションとFABが非表示になること

### 6-2. テーマ設定
- [ ] テーマセクションに「システム」「ライト」「ダーク」の3チップが表示されること
- [ ] 現在のテーマに対応するチップが選択状態であること
- [ ] チップタップで即座にテーマが切り替わること

### 6-3. モデレーション設定
- [ ] 「コンテンツモデレーション」セクションヘッダーが表示されること
- [ ] 成人向けコンテンツスイッチが正しいON/OFF状態で表示されること

- [ ] 各行に「非表示」「警告」「表示」の3チップが表示されること

## 7. 統合テスト

### 7-1. 画面遷移の正常系
- [ ] ホーム → メッセージ → 会話タップ → チャット → 戻る → 会話一覧に戻る
- [ ] プロフィール → 設定 → テーマ変更 → 戻る → プロフィール画面がテーマ適用済み
- [ ] ホーム → 検索 → 通知 → メッセージ → プロフィール → 各タブ正常表示

### 7-2. データ永続化
- [ ] テーマを変更 → アプリ終了 → 再起動 → テーマが保持されていること
- [ ] モデレーション設定を変更 → アプリ終了 → 再起動 → 設定が保持されていること

### 7-3. エッジケース
- [ ] ネットワーク未接続時のメッセージ画面 → エラー表示されクラッシュしないこと
- [ ] トークン失効時のAPI呼び出し → 自動リフレッシュ後にリトライされること
- [ ] 非常に長いメッセージテキスト → 表示が崩れず折り返されること
- [ ] 非常に長い表示名のユーザーとの会話 → 省略表示されること

### 7-4. パフォーマンス
- [ ] 会話一覧の高速スクロールでカクつきがないこと
- [ ] チャット画面のメッセージ100件以上でスムーズにスクロールできること
- [ ] テーマ切替時に画面のちらつきが最小限であること
